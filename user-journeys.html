Apollo · Content Stage Available

Turn one technical topic into a clear carousel.

Apollo's verified paths turn a topic into either a fixed seven-slide v1 carousel or an adaptive 6–10-slide v2 carousel, with themed, legible 1080×1350 PNG slides and no manual HTML editing.

Codex skill V1: seven slides V2: 6–10 slides Constrained HTML PNG output
Topic → skill → agents → JSON → HTML → PNG V1 and adaptive v2 generation, rendering, export, and deterministic validation are available.

Intended User Journey

1. Give Apollo a topic

A technical-content creator starts locally with one clear subject.

Available
  • Invoke apollo-generate for fixed v1 or apollo-generate-v2 for adaptive v2 content, for example ACID properties in databases.
  • Each generation entry point accepts exactly one required topic.
  • The reference style is concise, technical, diagram-led, and interview-oriented.

2. Get structured carousel content

A Codex custom agent compresses the topic into a teachable carousel before any pixels are rendered.

Available
  • V1 has its fixed seven-slide story; v2 selects 6–10 slides from teaching beats.
  • V2 uses a closed role sequence and bounded plain-text copy so each slide remains mobile-legible.
  • The artifacts are carousel-content.json for v1 and carousel-content-v2.json for v2.

3. Compose constrained themed HTML

A Codex custom renderer creates HTML within the repository-owned database theme assets and templates.

Available
  • The local database theme pack is derived from the reference style and provides the approved visual assets and templates.
  • Every slide renders at 1080×1350 pixels.
  • V1 has exactly seven sequential slides; v2 has 6–10 content-derived slides. Neither permits scripts, network access, or external assets.

4. Export shareable PNG slides

Browser rendering captures each slide exactly, without a per-pixel image-generation cost.

Available
  • Playwright screenshots each v1 or v2 slide independently.
  • V1 outputs slide-01.png through slide-07.png; v2 outputs the ordered files in slides-v2/.
  • All run artifacts stay in a local, inspectable run directory.

5. Trust the basic checks

The MVP catches mechanical failures before presenting the carousel as complete.

Available for v2
  • V2 checks structural content limits, slide count, image dimensions, ordered manifest paths, and browser overflow before publication.
  • This first release does not promise automatic research, citations, vision review, or publishing.
  • Accuracy review remains a human responsibility during the MVP.

6. Grow only after the baseline works

V1 and v2 are verified; 0004 remains the next Draft milestone.

Current Stage and Planned MVP Capabilities

Input

Start with one topic

Use the apollo-generate Codex skill with one topic. Extra options are deferred until defaults prove insufficient.

Content

Make a fixed or adaptive carousel

Generate concise structured copy for a fixed seven-slide v1 or adaptive 6–10-slide v2 carousel.

Render

Use one consistent visual system

Constrained agent-authored HTML uses one local database theme pack and reliable mobile dimensions.

Export

Save individual PNGs

Browser screenshots create one 1080×1350 image per slide in a local run folder.

Validate

Reject broken layouts

V2 verifies structural limits, dimensions, content-derived slide count, manifest paths, and text or page overflow before completing a run.

Scope

Stay local for now

An AI theme, theme plugins, retries, automatic citations, generated assets, and vision repair are post-MVP.

V1 produces request.json, validated carousel-content.json, HTML, PNGs, and a manifest. V2 separately produces request-v2.json, carousel-content-v2.json, index-v2.html, slides-v2/, and render-manifest-v2.json. Apollo has no shell CLI or direct LLM API runtime.
