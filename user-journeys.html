Apollo · Planned MVP

Turn one technical topic into a clear carousel.

Apollo is a planned local CLI for technical-content creators. Its first release will turn a topic into seven consistently themed, legible 1080×1350 PNG slides without manual HTML editing.

Intended User Journey

1. Give Apollo a topic

A technical-content creator starts locally with one clear subject.

Planned
  • Run apollo generate "ACID properties in databases".
  • The MVP keeps inputs intentionally small: the topic is required; slide count and theme use defaults.
  • The reference style is concise, technical, diagram-led, and interview-oriented.

2. Get a structured seven-slide story

A content pass compresses the topic into a teachable carousel before any pixels are rendered.

Planned
  • Slide roles are fixed: hook, context, teaching slides, example or interview tip, and recap.
  • Each slide has one primary idea and bounded copy so it remains mobile-legible.
  • The artifact is carousel-content.json.

3. Render fixed, themed layouts

A deterministic HTML/CSS renderer fills a small set of reusable layouts rather than generating arbitrary slides.

Planned
  • One initial theme and seven layouts provide consistent typography, spacing, diagrams, footer, and slide numbering.
  • Every slide renders at 1080×1350 pixels.
  • The renderer produces index.html and a render manifest.

4. Export shareable PNG slides

Browser rendering captures each slide exactly, without a per-pixel image-generation cost.

Planned
  • Playwright screenshots each .slide independently.
  • Outputs are named slide-01.png through slide-07.png.
  • All run artifacts stay in a local, inspectable run directory.

5. Trust the basic checks

The MVP catches mechanical failures before presenting the carousel as complete.

Planned
  • Checks confirm image dimensions, slide count, nonempty files, and browser overflow.
  • This first release does not promise automatic research, citations, vision review, or publishing.
  • Accuracy review remains a human responsibility during the MVP.

6. Grow only after the baseline works

The first three milestones prove the simple path before Apollo expands.

Draft sequence
  • 0001: carousel contract and content.
  • 0002: fixed renderer and PNG output.
  • 0003: validation and proof across five topics.

Approved MVP Capability

Input

Start with one topic

Use a local apollo generate "<topic>" command. Extra options are deferred until defaults prove insufficient.

Content

Make a seven-slide carousel

Generate concise structured copy suited to technical interview preparation, from hook through recap.

Render

Use one consistent visual system

Render fixed HTML/CSS layouts with a single initial theme, diagram-led teaching, and reliable mobile dimensions.

Export

Save individual PNGs

Use browser screenshots to create one 1080×1350 image per slide in a local run folder.

Validate

Reject broken layouts

Verify dimensions, slide count, nonempty output, and text or page overflow before completing a run.

Scope

Stay local for now

Posting, scheduling, extra themes, automatic citations, generated assets, and vision repair are post-MVP.

The renderer, CLI, and validation commands are planned—not available runtime capabilities yet. Apollo's first proof is five technical topics rendered without manual HTML edits.

Expected Run Artifacts

Request and content

request.json records the topic; carousel-content.json preserves the seven-slide narrative.

Render source

index.html, styles, and a render manifest make the deterministic result inspectable and rerenderable.

Slide images

slide-01.png through slide-07.png are the final carousel assets.

Validation result

A run report records whether dimensions, count, output files, and overflow checks passed.
