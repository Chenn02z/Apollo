Apollo · Content Stage Available

Turn one technical topic into a clear carousel.

Apollo turns a topic into an adaptive 7–10-slide carousel, with themed, legible 1080×1350 PNG slides and no manual HTML editing.

Codex skill 7–10 slides Constrained HTML PNG output
Topic → skill → agents → JSON → HTML → PNG Generation, rendering, export, and deterministic validation are available.

Intended User Journey

1. Give Apollo a topic

A technical-content creator starts locally with one clear subject.

Available
  • Invoke apollo-generate with a topic, for example ACID properties in databases.
  • The generation entry point accepts exactly one required topic.
  • The reference style is concise, technical, diagram-led, and interview-oriented.

2. Get structured carousel content

A Codex custom agent compresses the topic into a teachable carousel before any pixels are rendered.

Available
  • Content selects 7–10 slides from teaching beats.
  • A closed role sequence and bounded plain-text copy keep each slide mobile-legible.
  • The artifact is carousel-content.json.
  • Initial content and each review-requested candidate get up to three writer attempts. An approve_with_warnings or reject review can request up to two validated candidate rewrites; validation removes an invalid selected artifact, the last valid content is preserved until promotion, and an unavailable or invalid review does not block the run.

3. Compose constrained themed HTML

Deterministic local tooling expands validated content into the repository-owned database shell.

Available
  • The local database theme pack is derived from the reference style and provides the approved shell, assets, and templates.
  • Every slide renders at 1080×1350 pixels.
  • Each carousel has 7–10 content-derived slides and permits no scripts, network access, or external assets.

4. Export shareable PNG slides

Browser rendering captures each slide exactly, without a per-pixel image-generation cost.

Available
  • Playwright screenshots each slide independently.
  • The ordered files are stored in slides/.
  • All run artifacts stay in a local, inspectable run directory.

5. Trust the basic checks

The MVP catches mechanical failures before presenting the carousel as complete.

Available
  • Apollo checks structural content limits, slide count, image dimensions, ordered manifest paths, and browser overflow before publication.
  • This first release does not promise automatic research, citations, vision review, or publishing.
  • Up to three validated content reviews and two writer revisions improve publishability; initial content and each candidate get up to three writer attempts, unavailable or invalid review remains non-blocking, and human review is final.

6. Grow visual composition after the baseline

Content authoring, review, and fixed-shell rendering are verified; visual planning and constrained composition are next.

0001 and 0002 Verified
  • 0001-adaptive-carousel-content: verified writer, reviewer, bounded revision, and validated 7–10-slide content.
  • 0002-deterministic-fixed-shell-rendering-baseline: verified six-variant rendering and PNG export.
  • 0003-template-archive-and-carousel-art-direction: Accepted visual-planning stage; 0004-constrained-slide-composition remains Draft.

Current Stage and Planned MVP Capabilities

Input

Start with one topic

Use the apollo-generate Codex skill with one topic. Extra options are deferred until defaults prove insufficient.

Content

Make an adaptive carousel

Generate concise structured copy for an adaptive 7–10-slide carousel.

Render

Use one consistent visual system

Deterministically populated HTML uses one local database theme pack and reliable mobile dimensions.

Export

Save individual PNGs

Browser screenshots create one 1080×1350 image per slide in a local run folder.

Validate

Reject broken layouts

Apollo verifies structural limits, dimensions, content-derived slide count, manifest paths, and text or page overflow before completing a run.

Scope

Stay local for now

An AI theme, theme plugins, unbounded retry or repair loops, automatic citations, generated assets, and vision repair are post-MVP.

Apollo produces request.json, validated carousel-content.json, index.html, PNGs in slides/, and render-manifest.json; initial content and each review-requested candidate get up to three writer attempts, while up to three validated, versioned reviews may trigger two candidate content revisions, each promoted only after validation. Apollo has no shell CLI or direct LLM API runtime.
