Pipeline for One Creator

Apollo User Journeys

Apollo turns a topic or concept into a TikTok-ready technical education clip package through a local agent-driven pipeline — from ideation through script, timeline, and export. Below is how one creator moves through it end-to-end.

Topic-to-Clip Agent-Driven Local-First Verifiable
End-to-End topic → ideation → script → timeline → TikTok-ready clip, all through one pipeline

Agent and Workflow Flow

Apollo work moves through four anchored gates: $bootstrap → $requirements → $spec → $dev-loop → Verified. $context auto-triggers after settled decisions. $grill-with-docs pressure-tests from bootstrap and requirements. plan-next runs manually after a phase ships.

Inside $dev-loop, subagent flow is role-separated: explorer → implementer → test-runner → reviewer. The $test skill drives verification inside test-runner. $codex-agent-tracer records every step as durable evidence.

Handoff Artifacts — Inputs and Outputs

Every workflow handoff records the same fields so producers and consumers stay aligned.

Field Meaning
Producer Workflow or agent that created the artifact ($spec, implementer, etc.)
Consumer Next workflow or agent that reads it ($dev-loop, $context, etc.)
Artifact Path File path under docs/specs/, docs/milestones/, or .agent-trace/
Status One of Draft, Accepted, Implemented, or Verified — see Validation below
Settled Decisions Resolved scope, terminology, or architecture choices this handoff locks in
Unresolved Blockers Open questions explicitly deferred to child specs or later milestones
Next-Reader Docs Canonical docs the consumer should read (docs/ARCHITECTURE.md, etc.)
Agent Routing Log Which subagents were invoked, in what order, and any manual fallbacks
Trace Path .agent-trace/<workflow-id>/ containing trace.jsonl, graph.mmd, audit.md

Validation and Status Progression

Every artifact moves through a status pipeline with explicit gates. Skipping a gate requires a documented justification — deterministic mocked verification where the relevant spec requires it.

Stage Gates Pipeline Checks
Draft Initial creation; no gates passed N/A
Accepted spec-griller or $grill-with-docs pass; blocking questions resolved Ideation: {topic, angle} both non-empty and coherent
Script package: narration, audience, and duration fields present
Implemented implementer pass complete; architecture seams honored Timeline preservation: segment timing and visual instruction copy survive the pipeline
Deterministic mocked verification for implemented/scoped stages (no live external calls)
Verified test-runner pass + acceptance criteria satisfied; trace recorded. Reviewer is read-only audit that batches findings for the next implementation pass. Full pipeline regression: topic in, TikTok-ready clip package out

Failure-mode enforcement: if spec-griller/$grill-with-docs finds blocking ambiguity, the artifact stays Draft. If implementer is incomplete the artifact stays at Accepted or Implemented as appropriate. If tests fail the artifact stays Implemented (not Verified). Skipped gates prevent Verified unless a documented exception exists.

Skipped-gate policy: if a gate is bypassed (e.g. no spec-griller run), the handoff artifact must list the skip with a reason. Skipped-gate artifacts cannot be marked Verified until the gate runs.

  • Hands Draft milestones to $requirements only; may reshape or discard bare future milestones.

Apollo Pipeline Capabilities

Ideation

Generate topic and angle

Feed a raw topic or concept into apollo ideate and get structured output with topic, angle, and pipeline-ready metadata.

Scripting

Shape narration and audience

Transform ideation output into a script package with narration text, target audience, and duration constraints.

Timing

Preserve segment timing and visuals

Convert script to timeline with segment-level visual instructions. Timing and visual copy survive the full pipeline.

Rendering planned / 0004

Rasterize slideshow images

Planned milestone docs/milestones/0004-asset-driven-visual-render.md. Render 3-10 timeline segments into ordered 1080x1920 PNG slideshow images via deterministic HTML/CSS templates: exactly one image and manifest entry per segment, with first intro, middle content, and last ending roles and no duplicated intro or ending slides.

Export planned / 0005

Package slideshow image set

Planned milestone docs/milestones/0005-mvp-export-package.md. Package the PNG image set, manifest, and caption notes for manual TikTok slideshow upload.

Workflow

Role-separated agent pipeline

Use $bootstrap → $requirements → $spec → $dev-loop with subagent roles and status progression from Draft to Verified.

Audit

Trace every agent action

$codex-agent-tracer records delegations, edits, verifications, and reviews as .agent-trace evidence with Mermaid graphs.

Alignment

Keep context synchronized

$context auto-triggers after settled decisions to keep docs, skills, agents, and user-journeys.html coherent.

Apollo is strongest as a strict pipeline spine — ideation feeds script, script feeds timeline, timeline feeds the 0004 renderer, and the renderer feeds export. Bootstrap determines the MVP boundary; architecture seams keep future work clean; plan-next reshapes the next increment from real results.

Key Artifacts

Startup contract

AGENTS.md defines non-negotiable operating rules, auto-triggers $context after settled decisions, and records project-specific command gaps.

Product backbone

docs/PRODUCT.md and docs/CONTEXT.md capture Apollo’s purpose, scope, canonical terminology, and active constraints.

Architecture seams

docs/ARCHITECTURE.md documents current structure, approved seams, and deferred architecture. Updated by $context when seams open or close.

Specs and milestones

docs/specs/0001-0003 and docs/milestones/0001-0005 hold the implementation contracts and deliverable slices that drive Apollo forward.

Execution evidence

.agent-trace/ holds trace.jsonl, graph.mmd, and audit.md per workflow run — the durable evidence behind every handoff.
